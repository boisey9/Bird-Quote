# RFQ V2 Implementation Notes

## Guiding principle

The RFQ app is the customer-facing intake and workflow layer. It should capture structured intent, documents, and quote context. It should not become the final engineering configurator.

## V2 capability coverage

- RFQ Submission
- RFQ Tracking foundation
- Document Upload UI
- Quote History Access / Past Order Reference
- Bid Contract upload/context
- Edit quote feature foundation
- Prefilled dealer information
- Duplicate Quote foundation
- Structured Data Capture
- RFQ ID Generation placeholder
- Assignment/Queue/SLA foundation for backend
- Alerts/Notifications foundation
- Pipeline visibility foundation
- Internal dashboard foundation
- RFQ status management foundation
- Ownership management foundation
- Role/access/audit/process governance foundation
- User/account management foundation
- Rules engine configuration placeholder

## Customer flow preserved

1. Company Information
2. Bus Specifications
3. Features & Options
4. Review & Submit

## Not included in V2 starter

- CPQ integration
- CRM integration
- public RFQ lead form
- Type C/D expansion
- Bus Builder launch
- data warehouse feed
- advanced quote versioning
